### 二叉树的顺序存储

二叉树遍历：
$~~~~~~$ 前中后序，实际上是把非线性结构线性化
$~~~~~~$ 给出含空格的遍历序列，可以构造一个树

线索二叉树：
$~~~~~~$ 在有左右子树指针的数的基础上，对所有叶的左右子树指针替换为 x 序遍历的前导后续指针
$~~~~~~$ n 结点有 n+1 空链域用于存前导\后续

> 表达式语法：波兰式（前缀表达式）\逆波兰式（后缀表达式）
> $~~~~~~$ 当遇到两个连续数字，用它们之前的运算符计算后，放入原位
> $~~~~~~$ 具体来说，用一个栈逐个读入表达式，如果栈顶和将入栈的表达式均是数字，则出栈一个数一个符进行运算，计算后重新进行入栈判断

重构二叉树：
$~~~~~~$给出 **_中序、后序_** 或 **_中序、前序_** 可以重构二叉树
$~~~~~~$相反地，只给先序和后续遍历结果无法唯一确定二叉树

> 证明：
> $~~~~~$ 对遍历结果长度作数学归纳法：易证，当遍历结果长度=2、3 且存在树同时满足先序后续时，结论成立
> $~~~~~$ 对 k+1 长度的遍历结果，如果存在树同时满足先序后序，则先序第一项和后续最后一项一定相同
> $~~~~~$ 此时把先序除第一项，后续除最后一项的部分视为对一个树的要求（假设至少存在一棵这样树）
> $~~~~~$ （这里实际上只是构造一部分会发生重复的树来证明会发生重复）
> $~~~~~$ 可见，此时的二节点树（root 确定）有两种方案（子树为左\右）
> $~~~~~$ 考虑子树的遍历结果也不唯一（归纳假设），从而给出的两个遍历结果不能唯一确定一颗二叉树

由先序序列和中序序列确定二叉树：

```cpp
///递归调用：
struct Node{
    int elem;
    Node* lch;
    Node* rch;
}

Node* LinkSubTree(Node* FroSeq,int FroLen,Node* MidSeq,int MidLen)
{
    Node* root = new Node;
    root=FroSeq[0];

    int LeftTreeSize,RightTreeSize;//遍历Mid直到root可得
    root.lch=LinkSubTree(FroSeq+1,LeftTreeSize,MidSeq,LeftTreeSize);
    root.rch=LinkSubTree(FroSeq+1+LeftTreeSize,RightTreeSize,MidSeq+1+LeftTreeSize,RightTreeSize);

    return root;
}
```

---

> 后序遍历的非递归算法：
> 栈中是一条从根节点到访问节点的路；每个节点都会进入栈；栈顶是当前所在，p 指向下一个要考虑的结点
> 每次循环，p、q、栈顶至少更新一个
> p 的探路总是先左，左路空再看右；
> q 是上一次访问的结点，用它来从叶回溯上去
> 打印时，p 指向被打印的结点，栈顶是被打印的节点的上级节点

## 2024 年 11 月 13 日

树的先根遍历、后根遍历：
可见实际上树转化的二叉树：对应节点的二叉树的左子树包含了结点在树中的所有子树，右子树包含了结点在树中的所有兄弟节点

> 从而，为什么树的先根遍历对应转换后的先序遍历？
> 树的先根是根-左一子树-左二子树-...,由于根节点没有兄弟，转化的二叉树中根节点没有右子树；对二叉树的先序遍历先遍历根节点，再左子树
> 在先序遍历左子树时，先遍历根，再左子树、再右子树：对应树的根节点、自左到右的子树们、自左到右的兄弟树，与先根遍历相同
>
> 为什么树的后根遍历对应转换后的二叉树的中序遍历？
> 树的后根遍历：先子树再自己（对于有兄弟的树，可见顺序是先子树、再自己、再兄弟）
> 二叉树的中序遍历：先左子树（树的子树）再自己再右子树（树的兄弟）
> 可见相同

## 2024 年 11 月 25 日

最小生成树构造：Prim&Kruskal

Prim：从任一点开始扩充点集直到全集
可以证明，这种策略不失最优解

重点：维护一个包含各节点的表
表中记录该点到点集的距离（稍后定义）和取得该距离时另一个顶点
距离定义为，对于和点集不相邻的，距离为正无穷；相邻的，定义为连接的边中的最短边

每次更新表：检查每个节点与新连接的点是否有连接，如果有，是否短于当前连接，如果短，则更新链接表，否则一概不更新。
记录每次加入的节点，在辅助表中的边

复杂度分析：设顶点个数 n
O(n2),找 n 次，每次在 n 个节点中找最近的

Kruskal：
构造一个 n 点无边图，从权值最小的边开始遍历边，将第一个遇见的，连接非联通子图的边加入图，直到全图连接

只需要遍历一次边即可，不需要反复遍历，因为如果一条边被遍历而没有被考虑，说明他连接的两点已经被连接，不再需要这条边（如果连上就成环了）。

复杂度分析：设边数 e
O(e)

关键活动即不按期完成就会影响整个工程完成的活动。
时间余量：活动（边）的最迟开始-最早开始；从而关键活动的时间余量为 0

求关键路径（最长路）算法：关键路径算法
从而求最长路->求关键活动->找时间余量为 0 的活动->找事件的时间余量为 0 的事件

有事件的最早、最晚开始时间递推公式；求出每个顶点的最早、最晚开始时间后，判断每个活动的最早开始和最晚结束（其实是说下一个活动的最晚开始）

AVL 平衡二叉排序树的插入、删除
插入：
从插入的结点位置向上回溯到首个出现不平衡的位置，根据末三个节点的相互位置分类讨论

删除：删除后先构造一般的二叉排序树，再平衡化

   目标是通过有限的平衡化操作，使新子树的高度等于原子树的高度，此时新的修改局限于本子树内，不会影响树外的平衡

复杂度计算：
O 表示：上界；Omega 表示：下界；θ 表示：确界
递归方程：以二分查找为例

NP 完全性理论
P 类问题：指复杂度小于等于多项式时间的问题
NP 类问题：不确定是否可以在多项式时间内求解，但是可以在多项式时间内验证
P=NP：是否所有 NP 问题可以在多项式时间内求解
NP-hard 问题：至少有 NP 难度的问题
NPC 问题：NP 问题中最难的问题，所有的 NP 问题可以在多项式时间内规约到 NPC 问题

主定理：用于求解递归式的复杂度
对不能直接应用的递归式，逐行写出合并代价 f(n)，求和计算

线性时间选择算法：求解第 k 大元素
递归求解，复杂度 T(n)=T(3n/4)+O(n)，这是在最坏情况下；可以得到 T(n)=O(n)
快排的思路，但是只递归求解可能包含第 k 大元素的序列；选择基准也不是随便选，而是选择每 5 个数的中位数，求这些中位数的中位数，从而保证小于/大于基准的数都至少有 n/4 个

最接近点对问题：用分治可以达到 O(n)
找基准：(一维)最大值与最小值的平均数，记录离基准最近的两侧点；
（二维）一条直线 x=l，在求出左右的最小值 d 时，考虑 x=l-d 到 x=l+d 间的点；可以证明，在这个区域里至多只能有 2*（H/d * 2+2）个点

## 线性规划 Linear Programming

![问题形式](image-93.png)

单纯型法：考虑最优解总是出现在顶点上
![线性规划基本定理](image-94.png)
这里展开细讲：一共有 n 个变量，m 个线性表达式<=/>=/==0 的约束，n 个 xi>=0 的约束
由此，单纯型解中，至多有 m 个非零取值。
要确定 Rn 上一个点，需要给定这 n+m 个约束中 n 个约束作为方程组求解
基本定理指出，如果有最优解，则总有一基本可行最优解,即总有一个最优解是 n 个约束以等号成立时的取值
如果常数 b 为负，则乘-1；如果不为等号约束，加入松弛变量(根据需要决定松弛变量的符号)

具体策略：
测试若干约束组合，查看目标函数值: 最终目标是把目标函数写成 G=常数-(非负的非基本变量求和)，此时易见，目标函数在非基本变量全为 0 时取得最大值
线性规划问题的标准形式：只有等式约束和非负约束

> 单纯形法在将非基变量（入基变量）的值从 0 增加到某个正数时，必须确保：
> 其他基变量的值不会因为这次调整而变为负数（否则违反非负约束）。

> 离基变量的选择决定了入基变量能增长的最大值，而其他约束不会因此被破坏:实际上，由于上一次的解是非基变量全为 0，这里就在考虑将目标函数在离基变量减小的过程中，增大入基变量带来的增益的最大可能值(在离基变量不减少到负数的前提下)
> 最小比值测试的作用：
> 计算每个约束对入基变量增长的限制，找到最严格的限制（即最小的允许增长值）。
> 这个最严格的限制对应的基变量就是“受到限制最多”的变量，也就是离基变量。

考虑特殊标准形式：约束标准型问题(对变量的系数和分布有要求)

对于非等式约束:原变量->加入松弛变量
对于不存在 I 矩阵:加入人工变量保证参数矩阵存在 I(指的是存在每行一个仅在本行出现的变量且系数为正)

如果 max Z'=-(z1+z2)=0，则说明可能存在可行解
